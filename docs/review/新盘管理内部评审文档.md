## 1. 项目概述

### 1.1 模块定位

新盘管理模块是新房AI作业平台的**核心入口模块**，负责新盘信息的结构化录入、审核流程管理、探盘模板配置、上下线管理。

### 1.2 业务价值

| 价值点 | 说明 |
|-------|------|
| 信息结构化 | 81字段标准录入，确保楼盘信息完整度≥95% |
| 流程规范化 | 审核机制保证数据质量，审核通过率≥90% |
| 效率提升 | 模板配置效率≤5分钟/楼盘（使用模板） |
| 快速启动 | 上线后自动关联探盘任务、AI话术训练 |

## 2. 需求背景

### 2.1 问题痛点

| 问题 | 现状 | 期望 |
|-----|------|------|
| 信息分散 | 新盘录入没有统一标准，信息散乱不完整 | 81字段标准化录入 |
| 流程缺失 | 录入后无法关联后续流程 | 审核通过后自动进入配置/上线流程 |
| 质量无保证 | 缺少审核机制，数据质量无法保证 | 审核工作台，驳回需填写原因 |
| 效率低下 | 每次手动配置探盘模板 | 默认模板，≤5分钟/楼盘 |

### 2.2 目标指标

| 指标 | 目标值 | 衡量方式 |
|-----|-------|---------|
| 录入完成率 | ≥95%字段完整度 | 系统统计 |
| 审核通过率 | ≥90% | 系统统计 |
| 录入效率 | ≤15分钟/楼盘 | 操作计时 |
| 配置效率 | ≤5分钟/楼盘（使用模板）| 操作计时 |

## 5. 状态流转图

### 5.1 状态定义（7个状态，含1个过渡态）

| 状态 | 说明 | 可执行操作 | 用户感知 |
|-----|------|---------|---------|
| 草稿 | 录入中，未提交 | 编辑、提交审核、删除 | ✅ |
| 待审核 | 已提交，等待审核 | 审核（通过/驳回）、撤回 | ✅ |
| 审核通过 | 审核已通过，即将进入待配置 | — | ❌ 过渡态 |
| 待配置 | 等待配置探盘模板 | 配置模板、上线 | ✅ |
| 已上线 | 楼盘已上线，流程进行中 | 下线 | ✅ |
| 已驳回 | 审核驳回，需修改 | 编辑、重新提交、删除 | ✅ |
| 已下线 | 楼盘已下线，流程终止 | 上线（重新上线）、删除 | ✅ |

### 5.2 状态流转图

```
                    ┌─────────┐
                    │  草稿   │
                    └────┬────┘
                         │ 提交审核
                         ↓
                    ┌─────────┐
              ┌─────→│ 待审核  │←────┐
              │      └────┬────┘      │
              │           │ 审核通过   │ 驳回修改
              │           ↓            │
              │    ┌─────────────┐    │
              │    │ 待配置 │    │
              │    └──────┬──────┘    │
              │           │ 配置完成    │ 
              │           ↓            │
              │      ┌─────────┐      │
              │      │ 已上线  │      │
              │      └───┬─────┘      │
              │         │ 下线        │
              │         ↓            │
              │    ┌─────────┐      │
              └────│ 已驳回 │──────┘
                   └─────────┘
                         ↑
                   修改后重新提交
```

## 8. 验收标准

### 8.1 功能验收标准（P0）

| 功能点 | 验收条件（Given-When-Then） |
|-------|---------------------------|
| 信息录入 | Given 进入新建楼盘页面，When 填写必填字段并保存，Then 保存成功并提示 |
| 必填校验 | Given 填写信息时遗漏必填字段，When 点击保存，Then 提示缺失字段并高亮 |
| 图片上传 | Given 上传楼盘图片，When 选择图片文件，Then 图片显示上传进度，完成后预览可见 |
| 提交审核 | Given 楼盘信息完整，When 点击提交审核，Then 状态变为待审核，通知审核人员 |
| 审核通过 | Given 审核人员查看待审核楼盘，When 点击通过，Then 状态变为待配置 |
| 审核驳回 | Given 审核人员查看待审核楼盘，When 点击驳回并填写原因，Then 状态变为已驳回，通知录入人员 |
| 配置探盘模板 | Given 审核通过，When 配置探盘模板（路书/话术/录音/提醒），Then 保存成功 |
| 上线操作 | Given 审核通过/配置完成，When 点击上线，Then 发送通知给区域经理，楼盘变为已上线 |
| 下线操作 | Given 楼盘已上线，When 点击下线，Then 提示确认，发送通知，终止关联任务，楼盘变为已下线 |
| 下线通知 | Given 执行下线操作，When 完成，Then 通知区域经理和进行中的经纪人 |

### 8.2 业务规则验收

| 规则 | 验收条件 |
|-----|---------|
| 编辑权限 | 草稿/已驳回：普通录入人员可编辑；审核通过/已上线：**仅超级管理员** |
| 删除限制 | 审核通过后不可删除（包括已上线、已下线）；仅草稿/已驳回状态可删除 |
| 撤回规则 | 待审核状态可撤回，撤回后变为草稿 |
| 回收站 | **不设置回收站**，删除操作直接永久删除 |
| 通知触发 | 区域经理收到通知后才能在小程序侧看到楼盘 |
