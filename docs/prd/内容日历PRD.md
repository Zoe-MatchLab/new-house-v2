## 一、列表页

### 1.1 任务列表定位

- **执行入口**：查看被分配的任务，点击执行
- **创建入口**：创建各种类型的任务

### 1.2 任务状态

**执行的任务状态**：待开始、进行中、已完成、已超时、已停止、已拒绝

**创建的任务状态**：草稿、待开始、进行中、已完成、已超时、已停止

### 1.3 任务类型

（详见 PRD 原文）

### 1.4 任务卡片显示

- 主卡片支持跳转任务详情页
- 子任务支持跳转对应的执行页
- 点击任务卡片（待开始）→ 状态变为"进行中"
- 执行过程自动保存，支持中断后继续
- 完成文案生成后 → 任务完成
- 任务被停止 → 所有执行人任务变为"已停止"

### 1.5 筛选条件

- 状态筛选：全部、待开始、进行中、已完成、已超时、已停止、已拒绝
- 注意：执行人视角的任务列表不包含"草稿"状态（草稿仅在创建任务管理中可见）

### 1.6 创建入口与权限

- 不是所有用户都有创建任务的功能
- 按组织架构或与用户关联（后台手动配置权限）
- 总部上线新楼盘信息时，会同步到每个拥有创建任务权限的用户
- 可配置新房上线的通知人员
- 收到通知人员去创建任务时，会校验是否有任务创建权限

### 1.7 停止任务

**后台**：
- 目前只支持"每日选题类型"的停止操作，其他任务类型没有权限
- 需要 UI 在设计的时候进行增加

**小程序侧**：
- 停止按钮在"创建任务管理"文档中
- 按钮在原型上页面没有示例，只有文本描述

### 1.8 任务拒绝

- 在创建任务的时候，支持配置允许拒绝
- 开启后则用户可以拒绝
- 任务执行人接收到任务 → 任务状态：待开始
- 任务执行人点击拒绝任务 → 任务状态：已拒绝

## 三、执行任务

### 3.1 探盘任务执行

任务顶部会显示：任务名称、楼盘信息、楼盘政策、任务结束时间、任务进度

主区域按照执行阶段分 3 个子任务进行：
1. 探盘路书
2. 文案内容生成
3. 图片/视频生成

#### 阶段一：探盘路书

**描述**：探盘路书需要按照探盘任务创建时候的配置进行资料收集

**流程**：
- 进入前判断是否有打卡范围限制：有则校验，校验不通过需提示用户；没有则直接进入
- 点位收集顶部固定展示 已完成任务 3/5 必打卡点位 3/3，数字随配置和资料上传动态更新

**点位采集页面**：
- 按照创建任务时的点位配置进行资料上传
- 上传类型：文本、图片、语音、视频、标签（待定）
- 其中**图片需要 AI 审核**（前提是创建任务的时候 AI 审核配置开启，默认是关闭的），审核失败需要给用户提示
- 所有资料提交前支持预览、编辑、删除；提交后仅支持预览
- **必打卡点位需要全部资料上传完，并审核通过，才允许提交素材**
- 提交的时候需要客户二次确认；确认后不可更改，只可预览

> ⚠️ 点位收集的过程允许用户退出，用户上传的资料自动更新保存（不做提交），提交确认前用户可随时进入进行资料更新

**探盘路书单独做一个功能在工具入口点击进去？** - 待确认
（前提：只能选择已有新盘同时配置好探盘配置）

#### 阶段二：文案内容生成

支持多渠道内容的生成：小红书、公众号、视频号、朋友圈、转发语、推广话术

**视频号生成特殊处理**：
- 选择视频号渠道后，需要联动选择【闪电生成、普通生成、大师生成】
- 普通生成需要联动选择视频模版（必选）、用户声音（非必选）
- 大师生成联动选择视频模版（必选）、用户声音（非必选）、任务形象（非必选）

**生成模式说明**：
| 模式 | 说明 |
|------|------|
| 闪电生成 | 基于新盘的视频资料库，随机选取一个视频给到用户；立即生成 |
| 普通生成 | 基于新盘的视频资料库，随机选取一个视频，经过混剪 + 使用用户上传的声音生成视频声画组合为新的视频，发送给客户 |
| 大师生成 | 基于新盘的视频资料库，随机选取一个视频，允许用户自定义上传声音、人物形象，基于用户的声音、人物形象、阶段一的资料进行视频生成 |

> ⚠️ 用户确认完文案后即表示探盘路书任务完成（原因：图片、视频生成较慢）
> - 确认后不可修改，只允许查看

#### 阶段三：图片/视频生成

基于阶段二确认的渠道，为用户生成对应渠道的图片/视频内容
- 支持加入素材库、重新生成、下载等操作
- 确认后不可修改，只允许查看

## 四、任务详情

任务执行的详细信息查看，**不可编辑**

和创建任务的详情页共用，区别在于：
- 配置信息：不展执行人、提醒时间、通知人等不相关信息
- 执行情况：只展个人的执行情况
- 不支持催办
