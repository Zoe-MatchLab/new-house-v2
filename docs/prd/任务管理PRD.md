## 1. 背景与目标

### 1.1 模块定位

任务管理模块是新房AI作业平台的**统一任务中心**，管理所有前台产生的任务。

**重要说明**：原"探盘任务"模块升级为"任务管理"模块，探盘任务只是任务类型之一。

### 1.2 需求背景

**为什么要做任务管理？**
- 需要统一管理各类任务（探盘任务、话术训练、录音分析等）
- 区域经理需要移动办公查看团队任务进展
- 任务数据需要按参与关系进行权限控制

**当前问题**
- 任务分散在各个模块，无统一管理入口
- 移动端缺少团队视图，管理者无法查看团队任务
- 数据权限不清晰，存在信息泄露风险

### 1.3 目标

| 目标 | 说明 |
|-----|------|
| 统一任务管理 | PC后台管理所有任务的生命周期 |
| 移动任务执行 | 小程序端执行探盘任务（本期） |
| 任务级数据权限 | 基于任务参与的数据可见性，确保信息安全 |
| 支持多任务类型 | 本期探盘任务，后续可扩展话术训练、录音分析等 |

## 3. 任务类型定义

### 3.1 任务类型

| 任务类型 | 说明 | 状态 | 优先级 |
|---------|------|------|--------|
| 探盘任务 | 包含路书执行+话术训练+录音分析 | 本期 | P0 |
| 话术训练任务 | AI对话练习 | 本期 | P0 |
| 录音分析任务 | 录音上传+AI分析 | 本期 | P0 |
| 文案生成任务 | 内容自动生成 | 本期 | P0 |
| 自定义任务 | 用户自定义创建的任务 | 本期 | P0 |

### 3.2 探盘任务子任务结构

探盘任务包含三个**可选并行**的子任务：

| 子任务 | 说明 | 完成状态 | 产出数据 |
|-------|------|---------|---------|
| 路书执行 | 探盘路线执行、内容生成 | 未开始/进行中/已完成 | 内容记录 |
| 话术训练 | AI话术训练 | 未开始/进行中/已完成 | 训练记录/评分 |
| 录音分析 | 录音上传、AI分析 | 未开始/进行中/已完成 | 录音记录/分析结果 |

#### 子任务关系

| 关系 | 说明 |
|-----|------|
| 并行 | 三个子任务可以任意顺序完成，无依赖 |
| 可选 | 模板配置决定包含哪些子任务 |
| 独立状态 | 每个子任务有独立的完成状态 |

## 4. 任务状态定义

### 4.1 任务维度区分

任务分为两个维度：

| 维度 | 视角 | 说明 |
|-----|------|------|
| 创建人任务 | 创建者 | 受所有下发任务状态影响 |
| 执行人任务 | 执行人 | 每个执行人的独立任务 |

### 4.2 创建人视角状态（PC端/小程序创建人看）


| 状态 | 说明 |
|-----|------|
| 草稿 | 未下发 |
| 进行中 | 已下发，只要有≥1个执行人在进行中 |
| 已完成 | **所有执行人都完成任务** |
| 已超时 | 有≥1个执行人超时 |
| 已停止 | 被停止 |

**状态聚合规则**：
```
进行中：≥1个执行人进行中
已完成：所有执行人都完成
超时：≥1个执行人超时
已停止：≥1个执行人被停止
```

### 4.3 执行人视角状态

| 状态 | 说明 |
|-----|------|
| 待执行 | 下发后，还未开始执行 |
| 进行中 | 开始执行后 |
| 已完成 | 自己完成所有子任务 |
| 已超时 | 自己超时 |
| 已停止 | 被停止 |

### 4.4 状态流转

**创建人视角**：
```
                    ┌─────────┐
                    │  草稿   │
                    └───┬─────┘
                        │ 下发
                        ↓
                   ┌─────────┐
                   ┌──→│ 进行中  │←──┐
                   │      └───┬─────┘    │
                   │          │           │ 有执行人超时
                   │          │           ↓
                   │          │      ┌─────────┐
                   │          │      │ 已超时  │
                   │          │      └─────────┘
                   │          │
                   │          │ 所有执行人完成
                   │          ↓
                   │     ┌─────────┐
                   │     │ 已完成  │
                   │     └─────────┘
                   │
                   └───── 已停止
```

**执行人视角**：
```
                ┌─────────┐
                │ 待执行  │
                └───┬─────┘
                    │ 开始执行
                    ↓
               ┌─────────┐
               │ 进行中  │
               └───┬─────┘
                   │ 提交完成  │ 超时
                   ↓           ↓
              ┌─────────┐ ┌─────────┐
              │ 已完成  │ │ 已超时  │
              └─────────┘ └─────────┘
```

### 4.4 执行人变更的状态变化

| 操作 | 执行人任务状态 |
|-----|--------------|
| 移除执行人 | → 已停止 |
| 新增执行人 | → 进行中 |

## 5. 任务操作定义

### 5.1 PC管理后台操作（只读）

| 操作 | 说明 |
|-----|------|
| 查看 | 查看任务详情 |

### 5.2 小程序侧操作

| 操作 | 权限 | 说明 |
|-----|-----|------|
| 查看 | 所有人 | 查看任务详情 |
| 开始执行 | 执行人 | 待执行 → 进行中 |
| 提交完成 | 执行人 | 进行中 → 已完成 |
| 催办 | 仅创建人 | 进行中状态可催办 |
| 编辑 | 仅创建人 | 可编辑：时间 + 人员 |

### 5.3 编辑限制

| 可编辑项 | 说明 |
|---------|------|
| 截止时间 | 不能早于当前时间 |
| 执行人员 | 新增/删除执行人 |

## 6. 催办等级

### 6.1 催办等级定义

| 催办等级 | 触发时机 | 优先级 |
|---------|---------|--------|
| 轻度 | 24h前 | 重要 |
| 中度 | 12h前 | 重要 |
| 重度 | 2h前 | 紧急 |

### 6.2 配置规则

| 规则 | 说明 |
|-----|------|
| 用户可配置 | 用户可自定义配置时间 |
| 前置条件 | 重度必须配置了轻度和中度 |
| 数值要求 | 轻度 > 中度 > 重度 |
| 数据类型 | 大于0的整数（分钟） |

## 7. 权限模型

### 7.1 权限设计原则

> **核心原则：任务级数据可见性**
> 
> 只有任务创建者或任务参与者才能查看任务数据

### 7.2 数据可见性规则

| 用户类型 | 任务创建 | 任务可见范围 |
|---------|---------|------------|
| 超级管理员 | ✅ | 全部任务数据 |
| 区域经理 | ✅ | **仅自己创建的任务**数据 |
| 经纪人 | ❌ | **被分配的任务**数据 |
| 外部用户 | ❌ | 无任务创建权限（后续版本可能开放） |

### 7.3 权限矩阵

| 操作 | 超级管理员 | 区域经理 | 经纪人 |
|-----|-----------|---------|--------|
| 创建任务 | ✅ | ✅ | ❌ |
| 查看任务 | ✅ | 仅自己创建 | 仅被分配 |
| 催办 | ✅ | ✅（创建的任务） | ❌ |
| 编辑任务 | ✅ | ✅（创建的任务） | ❌ |

### 8.3 任务列表功能

| 功能 | 说明 |
|-----|------|
| 任务类型筛选 | 探盘任务/话术训练任务/录音分析任务/文案生成任务/自定义任务 |

| 创建人筛选 | 看自己创建的任务 |
| 搜索 | 按楼盘名称/执行人搜索 |

### 8.3 表格展示

| 任务名称 | 楼盘 | 创建人 | 执行人 | 任务状态 | 操作 |
|---------|-----|-------|-------|---------|------|
| 探盘任务A | 万科城市花园 | 区域经理A | 经纪人1, 经纪人2, +3 | 进行中 | 查看 |

**说明**：
- 一行 = 一个任务
- 执行人超过3人时折叠展示，如"经纪人1, 经纪人2, +3"
- 点击可展开查看全部执行人
- 任务状态按聚合规则计算（见4.2）
