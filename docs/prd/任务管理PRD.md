## 1. 背景与目标

### 1.1 模块定位

任务管理模块是新房AI作业平台的**统一任务中心**，管理所有前台产生的任务。

**重要说明**：原"探盘任务"模块升级为"任务管理"模块，探盘任务只是任务类型之一。

### 1.2 需求背景

**为什么要做任务管理？**
- 需要统一管理各类任务（探盘任务、话术训练、录音分析等）
- 区域经理需要移动办公查看团队任务进展
- 任务数据需要按参与关系进行权限控制

**当前问题**
- 任务分散在各个模块，无统一管理入口
- 移动端缺少团队视图，管理者无法查看团队任务
- 数据权限不清晰，存在信息泄露风险

### 1.3 目标

| 目标 | 说明 |
|-----|------|
| 统一任务管理 | PC后台管理所有任务的生命周期 |
| 移动任务执行 | 小程序端执行探盘任务（本期） |
| 任务级数据权限 | 基于任务参与的数据可见性，确保信息安全 |
| 支持多任务类型 | 本期探盘任务，后续可扩展话术训练、录音分析等 |

## 3. 任务类型定义

### 3.1 任务类型

| 任务类型 | 说明 | 状态 | 优先级 |
|---------|------|------|--------|
| 探盘任务 | 包含路书执行+话术训练+录音分析 | 本期 | P0 |

### 3.2 探盘任务结构

```
探盘任务
├── 基础信息

│   ├── 执行人（经纪人）
│   └── 截止时间
├── 探盘内容
│   ├── 路书执行（探盘路线、重点介绍）
│   ├── 话术训练（训练题目、题库）
│   └── 录音分析（录音配置、评分规则）
├── 打卡记录
│   ├── 定位打卡
│   └── 点位采集（照片/视频）
└── 完成状态
    ├── 已接受/已拒绝
    ├── 执行中
    └── 已完成
```

## 4. 权限模型

### 4.1 权限设计原则

> **核心原则：任务级数据可见性**
> 
> 只有任务创建者或任务参与者才能查看任务数据

### 4.2 数据可见性规则

| 用户类型 | 任务创建 | 任务可见范围 |
|---------|---------|------------|
| 超级管理员 | ✅ | 全部任务数据 |
| 区域经理 | ✅ | **仅自己创建的任务**数据 |
| 经纪人 | ❌ | **被分配的任务**数据 |
| 外部用户 | ❌ | 无任务创建权限（后续版本可能开放） |

### 4.3 权限矩阵

| 操作 | 超级管理员 | 区域经理 | 经纪人 |
|-----|-----------|---------|--------|
| 创建任务 | ✅ | ✅（PC后台） | ❌ |
| 分配任务 | ✅ | ✅ | ❌ |
| 执行任务 | ✅ | ✅ | ✅（小程序） |
| 查看任务详情 | ✅ | 仅自己创建 | 仅被分配 |
| 任务统计 | ✅ | 仅自己创建的任务 | ❌ |

### 4.4 关键洞察

> **"创建任务 ≠ 获得团队所有数据"**
> 
> 区域经理创建一个任务后，只能看**这个特定任务**的数据
> 
> 想看团队所有数据？需要**逐个创建任务**并关联成员

## 6. 小程序端功能

### 6.1 页面清单

| 页面 | 功能 |
|-----|------|
| 任务列表 | 我的任务、执行入口 |
| 任务详情 | 查看任务要求、执行操作 |
| 路书执行 | 探盘路线展示、定位打卡 |
| 点位采集 | 照片/视频拍摄 |
| 话术训练 | AI对话练习 |
| 录音分析 | 录音上传、分析结果 |

### 6.2 任务执行流程（探盘任务）

```
收到任务通知
    ↓
查看任务详情（楼盘信息、路书配置）
    ↓
接受/拒绝任务
    ↓
执行探盘
    ├── 定位打卡
    ├── 点位采集（样板间/沙盘/园林）
    ├── 话术训练（AI对话）
    └── 录音分析（录制话术）
    ↓
提交完成
```

### 6.3 团队视图（组织内用户可见）

| 用户类型 | 可见内容 |
|---------|---------|
| 区域经理 | 自己创建的任务列表、团队执行情况 |
| 经纪人 | 仅自己的任务 |

> **说明**：团队视图展示的是"我创建的"任务数据，不是"团队所有"数据

## 7. 状态流转

### 7.1 任务状态

| 状态 | 说明 | 可执行操作 |
|-----|------|---------|
| 待接受 | 任务已分配，等待执行人确认 | 接受/拒绝 |
| 执行中 | 任务已接受，正在执行 | 查看进度/提交完成 |
| 已完成 | 任务已提交，区域经理确认 | 查看 |
| 已超时 | 任务超过截止时间未完成 | 重新分配/延期 |

### 7.2 状态流转图

```
                    ┌─────────┐
                    │ 待接受  │
                    └───┬─────┘
                        │ 接受
                        ↓
                   ┌─────────┐
              ┌────→│ 执行中  │←────┐
              │      └───┬─────┘      │
              │          │ 提交完成   │ 逾期/重新分配
              │          ↓            │
              │     ┌─────────┐      │
              └─────│ 已完成  │──────┘
                    └─────────┘
```

## 8. 关联模块

### 8.1 任务管理与其他模块的关系

| 关联模块 | 关联方式 |
|---------|---------|
| 新盘管理 | 探盘任务关联楼盘，上线后区域经理收到通知 |
| 录音分析 | 探盘任务内包含录音分析子任务；PC端录音列表可查看任务关联录音 |
| 话术训练 | 探盘任务内包含话术训练子任务 |
| 消息通知 | 任务创建/分配/完成/超时 → 通知相关人员 |

### 8.2 数据关联说明

| 数据类型 | 关联方式 | PC端查看 |
|---------|---------|---------|
| 路书配置 | 关联新盘管理 | 可查看（只读） |
| 录音记录 | 关联任务ID | 录音分析模块查看（按任务筛选） |
| 话术训练记录 | 关联任务ID | 话术训练模块查看（按任务筛选） |

> **注意**：本期版本录音分析模块不主动关联任务，后续版本任务管理创建录音任务后会自动关联。

## 9. 用户类型与权限

### 9.1 用户类型

| 用户类型 | 来源 | 说明 |
|---------|------|------|
| 组织内用户 | 组织架构同步 | 手机号命中组织架构，可使用团队功能 |
| 外部用户 | 小程序注册 | 无组织关系，仅看个人数据 |

### 9.2 权限自动适配

| 用户类型 | 任务可见范围 | 团队视图 |
|---------|------------|---------|
| 组织内用户（经纪人） | 被分配的任务 | ❌ 不可见 |
| 组织内用户（区域经理） | 自己创建的任务 | ✅ 可见 |
| 外部用户 | 被分配的任务 | ❌ 不可见 |

> **无需切换**：系统根据用户类型自动确定上下文
