## 1. 背景与目标

### 1.1 模块定位

客户管理用于管理经纪人自己的客户，支持客户信息维护和跟进记录。

### 1.2 需求背景

**为什么要做客户管理？**
- 经纪人需要管理自己的客户资源
- 每次与客户的通话都会生成录音分析
- 需要将录音分析关联到具体客户，便于跟进

**关键场景**
```
经纪人对客户A多次通话
    ↓
每次通话生成录音分析
    ↓
录音分析关联客户A
    ↓
形成客户A的跟进记录
```

### 1.3 目标

| 目标 | 说明 |
|-----|------|
| 客户信息管理 | 新增/编辑/删除客户信息 |
| 客户跟进 | 关联录音分析，形成跟进时间线 |

## 3. 客户信息

### 3.1 基本信息

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| 客户姓名 | 文本 | 必填 |
| 手机号 | 文本 | 必填，唯一标识 |
| 意向楼盘 | 关联 | 可关联楼盘 |
| 客户状态 | 下拉 | 潜在/跟进中/成交/流失 |
| 备注 | 文本 | 备注信息 |

### 3.2 客户状态

| 状态 | 说明 |
|-----|------|
| 潜在 | 初步接触 |
| 跟进中 | 多次跟进中 |
| 成交 | 已成交 |
| 流失 | 流失客户 |

## 5. 小程序端功能

### 5.1 页面清单

| 页面 | 功能 |
|-----|------|
| 客户列表 | 我的客户列表 |
| 客户详情 | 客户信息+跟进记录 |
| 新增客户 | 添加新客户 |
| 编辑客户 | 修改客户信息 |
| 删除客户 | 删除客户 |

### 5.2 客户管理功能

| 功能 | 说明 |
|-----|------|
| 新增客户 | 添加客户信息 |
| 编辑客户 | 修改客户信息 |
| 删除客户 | 删除客户 |
| 客户筛选 | 按状态筛选 |

### 5.3 跟进记录

| 功能 | 说明 |
|-----|------|
| 跟进时间线 | 显示客户的所有跟进记录 |
| 关联录音 | 录音分析关联到客户 |
| 跟进状态 | 客户状态变更记录 |

### 5.4 客户绑定能力

在录音分析时，可选择绑定客户：
- 选择已有客户或新建客户
- 录音分析关联到客户
- 形成客户跟进时间线

```
录音分析 → 绑定客户 → 关联到客户跟进记录
```

## 6. 权限模型

### 6.1 数据可见性

| 端 | 用户类型 | 可见范围 |
|---|---------|---------|
| PC管理后台 | 超级管理员 | 所有客户 |
| 小程序端 | 经纪人 | 仅自己的客户 |
| 小程序端 | 区域经理 | 自己的客户 |

### 6.2 说明

- PC端客户管理为**超管专用**
- 小程序端用户只能管理自己的客户
- 客户手机号作为唯一标识
