## 1. 背景与目标

### 1.1 模块定位

客户管理用于管理经纪人自己的客户，支持客户信息维护和关联录音分析查看。

### 1.2 需求背景

**为什么要做客户管理？**
- 经纪人需要管理自己的客户资源
- 每次与客户的通话都会生成录音分析
- 需要将录音分析关联到具体客户，便于查看历史分析

**关键场景**
```
经纪人对客户A多次通话
    ↓
每次通话生成录音分析
    ↓
录音分析关联客户A
    ↓
在客户A详情页查看所有历史录音分析
```

### 1.3 目标

| 目标 | 说明 |
|-----|------|
| 客户信息管理 | 经纪人可新增/编辑/删除自己的客户 |
| 关联录音分析 | 一个客户可关联多条录音分析 |

### 1.4 本期不做

| 功能 | 原因 |
|-----|------|
| 客户状态维持 | 当前阶段复杂，暂不实现 |
| 意向楼盘关联 | 客户信息由前台小程序录入，暂不关联楼盘 |

## 3. 客户信息

### 3.1 基本信息

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| 客户姓名 | 文本 | 必填 |
| 手机号 | 文本 | 必填，用于关联录音 |
| 备注 | 文本 | 备注信息（可选） |
| 创建时间 | 时间 | 自动生成 |

### 3.2 说明

- **客户信息来源**：由小程序经纪人录入
- **意向楼盘**：暂不关联
- **客户状态**：暂不维护

## 4. PC端功能（管理后台）

### 4.1 页面清单

| 页面 | 功能 |
|-----|------|
| 客户列表 | 查看所有客户的录音分析关联情况 |
| 客户详情 | 查看客户基本信息及关联的录音分析列表 |

### 4.2 客户列表功能

| 功能 | 说明 |
|-----|------|
| 筛选 | 按经纪人 |
| 搜索 | 按客户姓名/手机号 |
| 查看 | 查看客户详情及关联录音 |

---

## 5. 小程序端功能

### 5.1 页面清单

| 页面 | 功能 |
|-----|------|
| 客户列表 | 我的客户列表 |
| 客户详情 | 客户信息+关联的录音分析列表 |
| 新增客户 | 添加新客户 |
| 编辑客户 | 修改客户信息 |
| 删除客户 | 删除客户 |

### 5.2 客户管理功能

| 功能 | 说明 |
|-----|------|
| 新增客户 | 添加客户姓名、手机号 |
| 编辑客户 | 修改客户姓名、备注 |
| 删除客户 | 删除客户（同时解除录音关联） |

### 5.3 录音分析关联

- 录音分析时可选择绑定客户
- 一个客户可关联多条录音分析
- 客户详情页展示关联的录音分析列表

---

## 6. 权限模型

### 6.1 数据可见性

| 端 | 用户类型 | 可见范围 |
|---|---------|---------|
| PC管理后台 | 超级管理员 | 所有客户 |
| 小程序端 | 经纪人 | 仅自己的客户 |

### 6.2 说明

- PC端客户管理为**超管专用**
- 小程序端经纪人只能管理自己的客户
- 客户手机号作为唯一标识

## 7. 关联模块
