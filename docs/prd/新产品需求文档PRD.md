## 1. 背景与目标

### 1.1 需求背景

新房AI作业平台是一款面向房地产经纪人的智能作业工具，通过AI能力提升经纪人专业能力和作业效率，同时为管理层提供数据支撑。

**现有问题**：
- 信息获取滞后：经纪人依赖线下踩盘
- 能力参差不齐：约看率、成交率不稳定
- 物料质量不一：内容无法保障
- 管理抓手缺失：缺乏数据支撑

### 1.2 目标

| 目标 | 衡量指标 |
|-----|---------|
| 快速了解楼盘 | 楼盘信息完整度≥95% |
| AI话术训练 | 经纪人覆盖率≥80% |
| 内容自动生成 | 生成效率提升50% |
| 数据驱动管理 | 管理层看板实时可见 |

## 3.9 用户管理与组织架构同步

### 3.9.1 用户来源

| 用户类型 | 来源 | 说明 |
|---------|------|------|
| 内部用户 | 组织架构同步 | 公司员工，通过组织架构同步 |
| 外部用户 | 自主注册 | 非公司员工，自主注册使用 |

### 3.9.2 当前问题

| 问题 | 说明 | 影响 |
|-----|------|------|
| **同步延迟** | 入职时组织架构同步不及时 | 新员工无法及时使用系统 |
| **离职未清理** | 员工离职后仍在组织架构中 | 已离职员工数据残留 |
| **状态不一致** | 手机号判断逻辑存在漏洞 | 员工离职后仍可登录 |

### 3.9.3 用户状态定义

| 状态 | 说明 | 可登录 | 可操作 |
|-----|------|-------|-------|
| 正常 | 在职员工，已同步 | ✅ | ✅ |
| 待同步 | 已同步但未激活 | ❌ | ❌ |
| 离职 | 已从组织架构移除 | ❌ | ❌ |
| 外部用户 | 自主注册 | ✅ | ✅ |

### 3.9.4 组织架构同步机制

#### 同步流程
```
组织架构数据
      ↓
  增量同步（定时/触发）
      ↓
  比对现有用户表
      ↓
  ┌─────────────────────────┐
  │  手机号命中           │
  ├─────────────────────────┤
  │ ✅ 在组织架构 → 标记正常 │
  │ ❌ 不在组织架构 → 标记离职 │
  └─────────────────────────┘
```

#### 同步策略

| 策略 | 说明 | 频率 |
|-----|------|------|
| 全量同步 | 完整比对组织架构 | 每日1次 |
| 增量同步 | 仅同步变更数据 | 每小时1次 |
| 实时触发 | 组织架构变更时触发 | 实时 |

### 3.9.5 离职处理机制

#### 自动处理
| 场景 | 处理方式 |
|-----|---------|
| 员工离职 | 自动标记为"离职"状态，限制登录 |
| 数据保留 | 保留历史数据（任务、录音、客户关联） |
| 角色移除 | 移除所有角色关联 |

#### 数据清理策略
| 数据类型 | 保留时长 | 说明 |
|---------|---------|------|
| 用户基本信息 | 永久 | 用于历史数据关联 |
| 任务执行记录 | 永久 | 业务数据 |
| 客户关联数据 | 永久 | 业务数据 |
| 录音文件 | 180天 | 合规要求 |
| Token/凭证 | 立即清除 | 安全 |

### 3.9.6 页面清单

| 页面 | 路径 | 功能 |
|-----|------|-----|
| 用户列表 | /pc/team/user-list.html | 用户搜索、筛选、状态管理 |
| 用户详情 | /pc/team/user-detail.html?id=xxx | 用户信息、同步记录 |
| 同步日志 | /pc/team/sync-log.html | 组织架构同步记录 |
| 离职管理 | /pc/team/former-list.html | 离职用户管理 |

### 3.9.7 验收标准

| 功能点 | 验收条件 | 优先级 |
|-------|---------|-------|
| 用户同步 | Given 新员工入职，When 组织架构同步，Then 自动创建用户并发送通知 | P0 |
| 离职标记 | Given 员工离职，When 同步时发现不在架构，Then 自动标记离职 | P0 |
| 登录限制 | Given 用户状态为离职，When 尝试登录，Then 提示账号已停用 | P0 |
| 数据保留 | Given 用户离职，When 查看历史数据，Then 仍可查看关联数据 | P0 |
| 同步日志 | Given 组织架构同步，When 查看日志，Then 显示同步详情和变更记录 | P1 |

### 3.9.8 技术实现

| 组件 | 说明 |

## 3.10 消息通知模块（PC端 + 小程序端）

### 3.10.1 功能概述

消息通知是跨模块的基础服务，为新盘管理、探盘任务等模块提供统一的通知触达能力。

| 功能 | PC端 | 小程序端 | 优先级 |
|-----|------|---------|-------|
| 通知列表 | ✅ | ✅ | P0 |
| 通知详情 | ✅ | ✅ | P0 |
| 通知设置 | ✅ | ✅ | P0 |
| 已读/未读 | ✅ | ✅ | P0 |
| 消息推送 | ✅ | ✅ | P0 |

### 3.10.2 通知类型定义

| 通知类型 | 触发场景 | 接收人 | 优先级 |
|---------|---------|-------|-------|
| 新盘上线通知 | 楼盘审核通过并上线 | 区域经理 | P0 |
| 新盘下线通知 | 楼盘下线操作 | 区域经理+进行中经纪人 | P0 |
| 探盘任务通知 | 任务创建/分配 | 经纪人 | P0 |
| 催办通知 | 任务即将到期 | 经纪人 | P1 |
| 审核结果通知 | 楼盘审核通过/驳回 | 录入人员 | P0 |
| 录音分析完成 | AI分析完成 | 经纪人 | P0 |
| 系统公告 | 平台通知 | 全部用户 | P1 |

### 3.10.3 通知触发规则

| 通知类型 | 触发条件 | 发送渠道 | 备注 |
|---------|---------|---------|-----|
| 新盘上线 | 楼盘状态变为"已上线" | 小程序+微信 | 区域经理才能看到该楼盘 |
| 新盘下线 | 执行下线操作 | 小程序+微信 | 同时终止关联任务 |
| 探盘任务 | 任务创建并分配 | 小程序+微信 | 包含楼盘信息和截止时间 |
| 催办通知 | 任务结束前24h/12h/2h | 小程序+微信 | 按配置级别发送 |
| 审核结果 | 审核通过/驳回 | 小程序+微信 | 驳回需显示原因 |

### 3.10.4 通知内容结构

| 字段 | 说明 |
|-----|-----|
| 标题 | 通知简述（如：新盘上线通知） |
| 内容 | 详细说明（含楼盘名称、操作时间等） |
| 关联类型 | 楼盘/任务/客户/录音等 |
| 关联ID | 关联数据的ID |
| 跳转链接 | 点击后跳转的页面 |
