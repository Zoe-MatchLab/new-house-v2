## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 当前经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- 新盘信息分散在各个渠道，管理人员无法统一管理
- 需要一个标准化的入口，将新盘信息结构化录入系统
- 新盘上线后需要快速启动探盘任务、AI话术训练、邀约录音等后续流程

**当前存在什么问题？**

- 新盘录入没有统一标准，信息散乱不完整
- 录入后无法快速关联到探盘任务、话术训练等后续流程
- 缺少审核机制，信息质量无法保证
- 缺少统一的探盘配置模板，区域经理每次需要手动配置

**不做会怎样？**

- 后续探盘任务、话术训练等模块无法正常流转
- 数据看板缺少楼盘基础数据支撑

### 1.2 目标

**做完之后期望达到什么效果？**

- 管理人员可以通过PC后台快速录入新盘信息
- 新盘信息结构化存储，支持后续流程关联
- 录入信息有审核机制，保证数据质量
- 审核通过后可快速配置探盘模板并上线
- 下线操作可安全终止关联任务

**如何衡量成功？**

| 指标      | 目标值         |
| ------- | ----------- |
| 新盘录入完成率 | ≥95%字段完整度   |
| 审核通过率   | ≥90%        |
| 录入效率    | 单个楼盘≤15分钟   |
| 配置效率    | 使用模板≤5分钟/楼盘 |

### 1.3 成功指标

| 指标    | 当前值    | 目标值           | 衡量方式 |
| ----- | ------ | ------------- | ---- |
| 录入时长  | 无标准    | ≤15分钟/楼盘      | 操作计时 |
| 字段完整度 | 参差不齐   | ≥95%          | 系统统计 |
| 审核效率  | 人工判断   | ≤5分钟/楼盘       | 系统统计 |
| 配置效率  | 每次手动配置 | ≤5分钟/楼盘（使用模板） | 操作计时 |

## 3. 功能概述

### 3.1 核心功能

新盘管理模块的**三大核心功能**：

| # | 核心功能         | 说明                    |
| - | ------------ | --------------------- |
| 1 | **管理新盘资料**   | 81字段结构化录入、图片上传、审核流程   |
| 2 | **配置默认探盘配置** | 路书内容、AI话术训练、录音配置、提醒时间 |
| 3 | **上线通知**     | 上线后发送通知给区域经理，触发探盘任务创建 |

### 3.2 功能流程

```
录入信息 → 保存草稿 → 提交审核 → 审核通过 → 配置探盘模板 → 上线
                              ↓                              ↓
                           驳回 → 修改 → 重新提交        发送通知给区域经理（定时/手动）
                                                                    ↓
                                                        区域经理创建探盘任务

下线时：
    ↓
下线操作 → 发送通知 → 终止关联任务 → 楼盘下线
```

### 3.3 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 探盘任务 | 上线后由区域经理创建，支持使用默认模板 |
| 话术训练 | 上线后自动关联训练题库 |
| 邀约录音 | 上线后自动关联录音任务 |
| 物料管理 | 上线后楼盘资料进入物料库 |
| 数据看板 | 上线后楼盘数据进入统计分析 |
| **消息通知** | 上线/下线时触发通知给相关人员 |

> **注意**：消息通知是跨模块基础设施，在新盘上线/下线时触发通知。

## 4. 业务规则

### 4.1 通用规则

| 规则   | 说明                                               |
| ---- | ------------------------------------------------ |
| 录入权限 | 仅新房项目维护人员可录入                                     |
| 审核权限 | 仅审核人员可审核                                         |
| 上线权限 | 审核通过后可上线                                         |
| 编辑权限 | 草稿/已驳回：普通录入人员可编辑；审核通过/已上线：**仅超级管理员**可修改；待审核：不可编辑 |
| 删除限制 | 审核通过后不可删除（包括已上线、已下线）；仅草稿/已驳回状态可删除                |
| 撤回规则 | 待审核状态可撤回，撤回后变为草稿                                 |
| 下线权限 | 仅总部管理员可下线                                        |
| 回收站  | **不设置回收站**，删除操作直接删除                              |

### 4.1.1 新建/编辑页面规则

| 规则   | 说明                                 |
| ---- | ---------------------------------- |
| 页面复用 | 新建与编辑共用同一页面，通过URL参数（?id=xxx）区分     |
| 标题显示 | 新建：显示"新建楼盘"；编辑：根据楼盘状态显示"编辑楼盘-{状态}" |
| 状态提示 | 编辑时显示当前楼盘状态，告知用户可进行的操作             |
| 必填校验 | 提交审核前必须校验必填字段，未填写时提示并高亮缺失字段        |
| 保存方式 | 草稿保存（不校验）/ 提交审核（校验必填）              |

### 4.1.2 楼盘详情页规则

| 规则   | 说明                        |
| ---- | ------------------------- |
| 按钮逻辑 | 根据楼盘状态显示不同操作按钮（见操作权限矩阵）   |
| 撤回入口 | 待审核状态显示"撤回"按钮，撤回后变为草稿状态   |
| 删除限制 | 审核通过/已上线状态不显示删除按钮         |
| 编辑跳转 | 点击"编辑"跳转编辑页，需校验权限（无权限时提示） |

### 4.2 探盘模板配置规则 ⭐新增

**默认模板包含配置项**：

| 配置项      | 说明               | 必填 |
| -------- | ---------------- | -- |
| 路书内容     | 探盘路线、重点介绍内容      | 是  |
| AI话术训练配置 | 训练题目类型、题库选择      | 否  |
| 邀约录音配置   | 录音任务模板、评分规则      | 否  |
| 任务结束时间   | 探盘任务截止时间         | 是  |
| 提醒（告警）配置 | 24h轻度/12h中度/2h严重 | 是  |

**配置流程**：

```
审核通过 → 选择是否配置探盘模板
              ↓是                    ↓否
         配置人员/超管配置     → 立即发送上线通知
              ↓
         保存模板
              ↓
         发送上线通知
```

**配置权限**：

- 待配置状态：超级管理员或配置人员可操作
- 已上线楼盘修改配置：无需审核，直接生效

**项目政策**：

- 非必填项
- 作为楼盘的额外说明信息

### 4.3 上下线规则 ⭐新增

**上线流程**：

| 场景        | 操作            | 结果             |
| --------- | ------------- | -------------- |
| 审核通过后立即上线 | 点击"上线"        | 楼盘上线，发送通知给区域经理 |
| 配置探盘后上线   | 先配置模板，再点击"上线" | 保存配置，楼盘上线，发送通知 |

**⚠️ 通知触发规则（重要）**：

| 角色   | 触发条件     | 可见范围         |
| ---- | -------- | ------------ |
| 区域经理 | 收到新盘上线通知 | 才能在小程序侧看到该楼盘 |
| 经纪人  | 被分配任务后   | 才能看到该楼盘的探盘任务 |

> **业务说明**：未收到通知前，区域经理在小程序侧看不到新上线的楼盘。通知是"可见"的触发条件。

**下线流程（高危）**：

| 步骤 | 操作                 |
| -- | ------------------ |
| 1  | 点击"下线"             |
| 2  | 调用API查询关联任务数（实时数据） |
| 3  | 系统显示确认弹窗（展示关联任务信息） |
| 4  | 确认下线               |
| 5  | 系统发送下线通知给区域经理+经纪人  |
| 6  | 自动终止该楼盘关联的进行中任务    |
| 7  | 楼盘状态变为"已下线"        |
| 8  | 区域经理小程序侧不再显示该楼盘    |

### 4.4 催办规则 ⭐新增

**自动催办触发条件**：

| 告警级别 | 触发时间      | 通知内容            |
| ---- | --------- | --------------- |
| 轻度   | 任务结束前24小时 | 提醒：任务即将到期       |
| 中度   | 任务结束前12小时 | 警告：任务即将到期，请尽快完成 |
| 严重   | 任务结束前2小时  | 紧急：任务即将到期，请立即处理 |

**手动催办**：

- 区域经理可在任务详情页点击"催办"按钮
- 催办消息发送至经纪人（小程序+微信）

**超时处理**：

- 超时不自动催办，仅发送超时通知
- 超时后任务标记为"逾期"

### 4.5 特殊场景/异常处理

| 场景       | 处理方式                 |
| -------- | -------------------- |
| 必填字段缺失   | 提示"请填写完整必填信息"，高亮缺失字段 |
| 图片上传失败   | 显示"上传失败"，提供重试按钮      |
| 重复楼盘     | 提示"该楼盘名称已存在"，是否继续创建  |
| 审核超时     | 超过48小时未审核，自动提醒审核人员   |
| 网络中断     | 自动保存草稿，恢复后可继续编辑      |
| 下线时有关联任务 | 提示"将终止X个进行中任务"，确认后执行 |

## 5. 新盘状态流转

### 5.1 状态定义

| 状态   | 说明            | 可执行操作      | 备注          |
| ---- | ------------- | ---------- | ----------- |
| 草稿   | 录入中，未提交       | 详情、编辑、提交审核、删除 | <br />      |
| 待审核  | 已提交，等待审核      | 详情、审核（通过/驳回）、撤回  | <br />      |
| 审核通过 | 审核已通过，待配置上线   | 详情、配置、上线    | 配置为可选操作     |
| 已上线  | 楼盘已上线，流程进行中   | 详情、下线        | <br />      |
| 已驳回  | 审核驳回，需修改      | 详情、编辑、提交审核    | <br />      |
| 已下线  | 楼盘已下线，流程终止    | 详情、编辑、配置、上线   | <br />      |

> **注意**：
> - 审核通过后不可删除
> - 已驳回状态不可删除
> - "配置"为可选操作，审核通过后可立即上线，也可先配置探盘模板再上线

> **注意**：状态定义共**7个**（含1个过渡态"审核通过"），其中"审核通过"是过渡态，用户几乎感知不到（瞬间进入待配置）。

### 5.2 状态流转图

```
                    ┌─────────┐
                    │  草稿   │
                    └────┬────┘
                         │ 提交审核
                         ↓
                    ┌─────────┐
              ┌─────→│ 待审核  │←────┐
              │      └────┬────┘      │
              │           │ 审核通过   │ 驳回修改
              │           ↓            │
              │      ┌─────────┐      │
              │      │ 审核通过 │      │
              │      │ (可配置) │      │
              │      └────┬────┘      │
              │         │ 上线       │
              │         ↓            │
              │      ┌─────────┐      │
              │      │ 已上线  │      │
              │      └───┬─────┘      │
              │         │ 下线        │
              │         ↓            │
              │    ┌─────────┐      │
              │    │ 已下线  │──────┘
              │    └─────────┘
              │         ↑
              │    修改后重新提交
              │         │
              └────┌─────────┐
                   │ 已驳回 │
                   └─────────┘
```
